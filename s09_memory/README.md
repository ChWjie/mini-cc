# s09_memory: 跨会话持久记忆

## 问题
上一章已经解决了上下文爆炸的问题，但 Agent 在多轮会话之间仍然缺少“长期记住用户偏好和项目事实”的能力。这样容易导致重复提问、忘记用户习惯，或在新会话中重新解释相同约束。

## 本章目标
本章新增一个持久化记忆系统，让 Agent 能把重要信息保存到磁盘，并在后续会话中根据上下文自动加载相关记忆。

## 数据流
用户对话 -> 识别关键信息 -> 写入 .memory/ 记忆文件 -> 后续对话时自动选择相关记忆并注入上下文。

## 新增代码
本章新增了以下内容：

- MEMORY_DIR 与 MEMORY_INDEX：用于保存和索引记忆文件
- write_memory_file()：把用户偏好、项目事实或参考信息写入记忆文件
- _rebuild_index()：重建 MEMORY.md 索引，便于快速查看可用记忆
- select_relevant_memories()：从最近对话中选择最相关的记忆
- load_memories()：把相关记忆内容注入当前上下文
- extract_memories()：从对话中自动提取新的记忆
- consolidate_memories()：当记忆量过多时进行合并和精简

## 相对上一章变化
相比 s08_context_compact，本章保留了：

- Agent Loop 的基本执行方式
- 上下文压缩与摘要机制
- 工具调用与子 Agent 能力

新增的内容是：

- 持久化记忆文件系统
- 跨会话记忆检索与注入
- 自动记忆提取与整合

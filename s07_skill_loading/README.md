# s07_skill_loading: 按需技能注入

## 问题
上一章已经具备了子 Agent 的拆分能力，但在真实场景中，Agent 仍然很难在有限上下文中同时持有大量专业知识。若一开始就把所有技能内容都塞进提示词，成本会很高，且很多技能并不会被马上使用。

## 本章目标
本章新增一种两层技能加载机制：

- 第一层：在系统提示词中先给出技能目录和简短描述
- 第二层：当模型真正需要时，再通过 load_skill 工具读取完整技能内容

## 数据流
用户输入 -> Agent 判断是否需要某个技能 -> 触发 load_skill -> 返回完整 SKILL.md 内容 -> Agent 根据技能内容继续执行任务。

## 新增代码
本章新增了以下内容：

- _parse_frontmatter()：解析 SKILL.md 的 YAML 头信息
- _scan_skills()：扫描 skills/ 目录，建立技能注册表
- list_skills()：生成简短的技能目录列表
- build_system()：把技能清单注入到 SYSTEM 提示词中
- load_skill()：按名称返回完整技能内容
- load_skill 工具 Schema：允许 Agent 在运行时请求完整技能内容

## 相对上一章变化
相比 s06_subagent，本章保留了：

- Agent Loop 的基本执行逻辑
- todo_write / task 等工具能力
- Hook 与权限控制机制

新增的内容是：

- skills/ 目录扫描与注册
- 系统提示词中的轻量级技能目录
- 按需加载完整技能内容的机制

## 运行与验收
运行方式：

```bash
python s07_skill_loading/code.py
```

建议输入：

```text
请帮我查看当前项目中有哪些可用技能，并在需要时加载相关的技能内容来完成任务。
```

预期结果：

- Agent 会先看到技能目录
- 在需要时调用 load_skill
- 获取完整技能内容后继续执行任务

实际验证：

- 章节代码已完成并集成到当前项目版本
- 可通过当前环境运行

# s05_todo_write: 任务规划与待办提醒

## 问题
上一章已经具备了 Hook 与权限控制能力，但在真实开发中，Agent 仍然容易在复杂任务中“边想边做”，缺少一个显式的任务规划机制。这样很容易导致执行过程漫无目的、步骤混乱，尤其是多轮对话时容易忘记前面的计划。

## 本章目标
本章新增一个 todo_write 工具，用于让 Agent 在开始复杂任务前先生成任务清单，并在执行过程中持续更新状态。同时引入一个提醒机制：如果连续多轮没有更新 todo，系统会自动提示 Agent 重新整理当前任务。

## 数据流
用户输入 -> LLM 生成工具调用 -> todo_write 工具执行 -> 当前任务列表更新 -> Agent 继续执行后续步骤。

整体流程如下：

1. 用户提出一个多步骤任务。
2. Agent 先把任务拆成多个待办项。
3. todo_write 将任务状态写入内存中的 CURRENT_TODOS。
4. 后续工具调用继续执行，直到任务完成。
5. 如果连续多轮没有刷新任务，系统自动注入提醒信息。

## 新增代码
本章新增了以下内容：

- CURRENT_TODOS：保存当前会话的任务状态
- _normalize_todos()：校验 todo 数据结构是否合法
- run_todo_write()：将任务列表更新到内存中，并打印当前任务
- todo_write 工具 Schema：允许 Agent 提交待办列表
- rounds_since_todo 计数器：用于触发 nag reminder

```bash
python s05_todo_write/code.py
```

建议输入：

```text
帮我先分析当前项目结构，然后写一个简单的 README 说明当前开发进度。
```

预期结果：

- Agent 会先使用 todo_write 生成任务列表
- 后续执行过程中会更新任务状态
- 如果长时间没有更新，系统会提示“Update your todos.”

# s08_context_compact: 上下文压缩与历史摘要

## 问题
上一章已经支持按需加载技能，但在真实长时间对话中，消息历史和工具输出会持续增长，最终导致上下文过大、API 调用失败，甚至触发 prompt_too_long。

## 本章目标
本章新增一套上下文压缩流水线，优先用低成本方式压缩上下文，必要时再调用更重的摘要机制，确保 Agent 能持续长时间运行。

## 数据流
用户输入 -> Agent 执行工具 -> 历史消息增长 -> 触发压缩流水线 -> 预算压缩/裁剪/占位/摘要 -> 继续执行。

## 新增代码
本章新增了以下内容：

- snip_compact()：裁剪中间消息，保留头尾上下文
- micro_compact()：把旧的工具结果替换为占位符
- persist_large_output()：把超大的工具输出持久化到磁盘
- tool_result_budget()：控制工具结果的上下文占用预算
- summarize_history()：调用模型对历史对话生成摘要
- compact_history()：把历史消息压缩成一段摘要
- reactive_compact()：当 API 返回超长提示词错误时进行紧急压缩
- compact 工具：允许 Agent 在运行时主动触发压缩

## 相对上一章变化
相比 s07_skill_loading，本章保留了：

- Agent Loop 基本执行方式
- 工具调用体系与技能加载能力
- Hook 与权限控制机制

新增的内容是：

- 上下文压缩与摘要机制
- 大工具输出的持久化保存
- 自动/手动压缩策略

## 运行与验收
运行方式：

```bash
python s08_context_compact/code.py
```

建议输入：

```text
请帮我连续执行一段较长的任务，并观察上下文压缩是否会自动触发。
```

预期结果：

- 长时间对话后会触发压缩流程
- 历史消息会被摘要替代，避免上下文爆炸
- 超大工具输出会被持久化保存或替换成占位符
